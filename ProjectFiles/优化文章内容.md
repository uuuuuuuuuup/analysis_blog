## 二、优化目标

### 2.1 格式排版优化

#### 2.1.1 报告结构标准化

设计统一的报告结构模板，确保所有报告风格一致：

```
# [公司名称]（[股票代码]）稳健投资策略分析报告

> 📅 分析日期：YYYY-MM-DD | 📊 数据截止：YYYY年度 | 📋 分析框架：稳健投资策略 v2.0

---

## 📋 因子 1A：五分钟快筛（一票否决）
## 🔍 因子 1B：深度定性分析
## 📈 因子 2：穿透回报率粗算
## 💰 因子 3：穿透回报率精算
## 🎯 因子 4：估值与安全边际
## 📝 最终决策输出
## 📎 附录
```

#### 2.1.2 视觉层次优化

- 使用分隔线区分大章节
- 使用 emoji 图标增强可读性
- 重要结论使用引用块高亮
- 计算过程使用代码块

| `{{< bar-chart >}}` | 柱状图对比 | Chart.js / ECharts |
| `{{< pie-chart >}}` | 结构占比饼图 | Chart.js / ECharts |
| `{{< radar-chart >}}` | 多维评分雷达图 | Chart.js / ECharts |

#### 2.3.2 图表应用场景

| 场景 | 图表类型 | 数据示例 |
|:---|:---|:---|
| 营业收入年度对比 | 柱状图 | 2020-2025年营收数据 |
| 净利润率趋势 | 折线图 | 5年净利率变化 |
| 资产结构分析 | 饼图 | 流动资产/固定资产/其他占比 |
| 护城河多维度评分 | 雷达图 | 规模效应/品牌壁垒/渠道壁垒等 |
| 穿透回报率 vs 门槛 | 仪表盘 | GG=4.90% vs II=3.81% |
| 现金流构成 | 瀑布图 | OCF → CAPEX → FCF |
| 敏感性分析 | 柱状图组 | 不同收入情景下的回报率 |

#### 2.3.3 技术实现方案

**方案一：Chart.js（推荐）**
- 轻量级，加载快
- 支持 Canvas 渲染
- 响应式设计友好
- CDN 引入即可

**方案二：ECharts**
- 功能更强大
- 中文文档完善
- 支持复杂图表（瀑布图、雷达图等）
- 体积较大

**方案三：纯 CSS + SVG**
- 无需外部依赖
- 加载最快
- 适合简单图表
- 复杂图表实现困难

**建议采用：Chart.js + CSS 组合方案**
- 简单图表用 CSS 实现
- 复杂图表用 Chart.js
- 保证性能和兼容性

## 六、踩坑经验记录

### 6.1 Hugo Front Matter 必需

**问题**：Hugo 构建报错 `invalid character '{' looking for beginning of object key string`

**原因**：Markdown 文件缺少 YAML front matter（文件头部的元数据）

**解决方案**：每个 Markdown 文件必须在开头添加 front matter：

```yaml
---
title: "公司名称（股票代码）稳健投资策略分析报告"
date: 2026-04-15
draft: false
---
```

**注意**：front matter 必须以 `---` 包围，且位于文件最开头

---

### 6.2 Shortcode 嵌套问题

**问题**：在 `decision-card` shortcode 内部嵌套 `financial-table` 时，页面显示 `</table>` 等原始 HTML 标签

**原因**：Hugo 的 shortcode 有两种语法：
- `{{< >}}` - 输出原始 HTML，不经过 Markdown 处理
- `{{% %}}` - 输出会经过 Markdown 处理

当父 shortcode 使用 `{{ .Inner | markdownify }}` 时，嵌套的 `{{< >}}` 语法会被转义成文本显示

**解决方案**：

**方案一（推荐）**：在父 shortcode 内部使用普通 Markdown 表格，不嵌套 shortcode

```markdown
{{< decision-card >}}
**【标题】**

| 项目 | 内容 |
|:---|:---|
| 数据1 | 值1 |
| 数据2 | 值2 |
{{< /decision-card >}}
```

**方案二**：将嵌套的 shortcode 改用 `{{% %}}` 语法（但可能导致其他问题）

```markdown
{{< decision-card >}}
{{% financial-table %}}
| 项目 | 内容 |
|:---|:---|
| 数据1 | 值1 |
{{% /financial-table %}}
{{< /decision-card >}}
```

**最佳实践**：避免 shortcode 多层嵌套，保持结构简单

### 6.3 代码块折叠处理

**问题**：报告中大量计算过程代码块影响阅读体验

**解决方案**：创建 `collapse` shortcode 实现可折叠代码块

**Shortcode 文件**：`layouts/shortcodes/collapse.html`

```html
<details class="code-collapse">
  <summary class="code-collapse-summary">{{ .Get "title" | default "展开详情" }}</summary>
  <div class="code-collapse-content">
    {{ .Inner | markdownify }}
  </div>
</details>
```

**使用方式**：

```markdown
{{< collapse title="📊 展开计算过程" >}}
\`\`\`
计算代码内容...
\`\`\`
{{< /collapse >}}
```

**CSS 样式**：在 `static/css/report-enhanced.css` 中添加折叠样式

### 6.5 优化报告的标准流程

1. **添加 front matter**：确保文件开头有正确的 YAML 元数据
2. **使用 report-header**：替换原有标题为 shortcode
3. **使用 section-divider**：为每个大章节添加分隔标题
4. **使用 screening-checklist**：快筛检查表使用卡片式布局
5. **使用 data-section**：为财务数据表格添加紫色渐变头部
6. **使用 conclusion-box**：为重要结论添加高亮框
7. **使用 collapse**：将计算过程代码块折叠
8. **使用 decision-card-v2**：最终决策部分使用新版卡片
9. **使用 factor-summary**：各因子结果使用摘要卡片

```
{{< section-divider icon="💰" title="因子 3：穿透回报率精算" >}}
...

{{< section-divider icon="🎯" title="因子 4：估值与安全边际" >}}

```

**要点**：
1. 报告开头直接展示「投资结论」章节
2. 包含：决策卡片 + 风险提示 + 后续跟踪事项
3. 详细分析放在「详细分析」章节下
4. 因子4步骤5不再重复展示结论，仅提示"结论已在报告开头展示"
5. **标题规范**：文章标题由 Hugo 主题根据 front matter 的 `title` 字段自动生成，`report-header` shortcode 不再生成 h1 标题，仅显示元数据（分析日期、数据截止、分析框架）
6. **分割线规范**：
   - `section-divider` shortcode 已自带分割线样式，其后方不需要再加 `---` 分隔线
   - `report-header` 已自带底部边框，其后方不需要再加 `---` 分隔线
   - 仅在以下场景使用 `---` 分隔线：
     - 主要章节之间（如"投资结论"与"详细分析"之间）
     - 因子末尾汇总表与下一个因子之间
     - 正文与附录之间
     - 附录与页脚信息之间
7. **表格 shortcode 规范**：
   - `financial-table` 和 `data-section` 功能相同，都用于展示数据表格
   - 内部直接使用 Markdown 表格语法，shortcode 会自动渲染
   - 不要在 shortcode 内部使用 `<table>` 标签

### 6.9 样式设计原则

**设计理念**：简约、高级、无"AI味"

**参考风格**：类似专业投资报告网站（如 terancejiang.com 的中国移动报告）

**核心原则**：

1. **无渐变**：使用纯色背景（#fafafa、#fff）
2. **无阴影**：仅用边框和分隔线
3. **极小圆角**：0.25rem（几乎无圆角）
4. **细边框**：1px solid，颜色 #e5e5e5 或 #eee
5. **小字号**：
   - 正文：0.7rem - 0.75rem（约11-12px）
   - 标题：0.8rem - 1rem
   - 标签/徽章：0.5625rem - 0.625rem
6. **状态指示**：左侧 2px 彩色边框 + 浅色背景
7. **表格列宽**：`width: fit-content` 自适应内容，不强制撑满

**颜色方案**：

| 用途 | 颜色 |
|:---|:---|
| 边框/分隔线 | #e5e5e5 / #eee |
| 背景灰 | #fafafa |
| 主文字 | #222 |
| 次文字 | #666 |
| 成功/通过 | #2e7d32 / #e8f5e9 |
| 警告/注意 | #f57c00 / #fff8e1 |
| 危险/失败 | #c62828 / #ffebee |
| 信息 | #1565c0 / #e3f2fd |

**表格样式要点**：

```css
/* 表格宽度自适应内容，避免右侧留白 */
.data-section-content table {
  width: fit-content;
  border-collapse: collapse;
  font-size: 0.75rem;
}

/* 表头小号大写字母 */
.data-section-content th {
  background: #fafafa;
  font-weight: 500;
  color: #666;
  font-size: 0.625rem;
  text-transform: uppercase;
  letter-spacing: 0.02em;
}
```

**避免的设计元素**：
- ❌ 渐变背景
- ❌ 阴影效果
- ❌ 大圆角（>0.25rem）
- ❌ 粗边框（>1px）
- ❌ 过大的字号（>1rem）
- ❌ 表格强制撑满容器
- ❌ 过多的 emoji
